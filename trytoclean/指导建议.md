# **① 实践线：每天构建一小步，可运行的“一个功能”**

比如你在做 RTSP server：

- 今天：解析 DESCRIBE
- 明天：构建 Session 管理器
- 后天：抽象 Transport
- 再一天：实现纯 UDP RTP 推流
- 再下一天：增加 TCP interleaved

要求是：

**每天可运行、可验证、有产出，不追求完美结构。**

动手的意义不是“写漂亮代码”，而是让你：

- 碰到真实问题
- 逼自己去查 RFC
- 逼自己去思考模块边界
- 逼自己理解流程
- 逼自己优化设计

这是提升工程能力最现实的方式。

------

# **② 抽象线：每天做一个“五分钟系统抽象练习”**

只需要 5 分钟，我来指导你每天做一个“小抽象训练”。
 例如：

### 练习 A：RTSP PLAY 从流程到组件

问自己 3 个问题：

1. PLAY 的输入和输出是什么？
2. PLAY 要操作哪些组件？（session、transport、packetizer）
3. 代码中这些组件具体是哪些类？

这样每天抽象一个点，你会很快形成“工程抽象脑”。

------

### 练习 B：模块统一法（你非常需要这个）

比如：

**怎样让 TCP interleaved 与 UDP 推流使用统一接口？**

你不需要代码
 只需要回答：

```
应不应该存在 ITransport？  
它应该有哪三个核心接口？  
每个接口负责什么？
```

五分钟即可
 但这个能力成长极快。

------

### 练习 C：流程拆解法

例如 RTSP：

```
OPTIONS → DESCRIBE → SETUP → PLAY → TEARDOWN
```

问：

- 每一步依赖什么状态？
- 每一步读写什么数据结构？
- 哪一步需要 session？
- 哪一步需要 transport？
- 哪一步要构建 SDP？

只要每天做一个流程
 一个星期后你的系统设计能力会提升非常明显。
