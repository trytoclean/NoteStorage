## 1.1 使用场景分析

### 1.1.1 有界队列（BoundedBlockingQueue）适用场景

**1.内存受限环境**

- 嵌入式系统或资源受限的环境
- 需要对内存使用有明确上限的场景

**2.生产者速度快于消费者**

- 当生产者可能在短时间内产生大量数据时
- 防止无限制地消耗内存，起到"背压"(backpressure)作用

**3.需要流量控制的系统**

- 限制上游系统的生产速率
- 保护下游系统不被过载

**4.实时系统**

- 对响应时间有严格要求的场景
- 避免因队列过长导致的处理延迟

**5.服务质量(QoS)保证**

- 需要丢弃或拒绝过量请求
- 优先处理重要任务时

### 1.1.2 无界队列（BlockingQueue）适用场景

**1.消费者速度快于或等于生产者**

- 当消费能力足够强，不会导致队列无限增长
- 数据生产速率相对稳定可预测

**2.不希望阻塞生产者**

- 生产者任务关键或高优先级
- 需要解耦生产者和消费者处理速率

**3.事件收集和日志系统**

- 短时间内可能有大量事件，但最终会被处理
- 不希望因队列满而丢失数据

**4.任务调度系统**

- 需要接收所有任务请求而不拒绝
- 后续可通过优先级或其他策略处理

**5.缓冲系统**

- 生产和消费速率不均衡但平均来看可以持平
- 需要吸收流量波峰波谷

## 1.2 设计差异

### 1.2.1 内存管理

**有界队列**

- 使用**固定大小的内存空间**（如我们实现的环形缓冲区）
- **预先分配所有内存(分配item)**，避免运行时分配

**无界队列**

- 使用动态增长的数据结构（如`std::deque`）
- 随需分配内存，可能导致频繁内存分配
- 理论上只受系统内存限制

### 1.2.2 阻塞行为

**有界队列**

- **双向阻塞**：队列满时生产者阻塞，队列空时消费者阻塞
- 需要两个[条件变量](https://zhida.zhihu.com/search?content_id=257200624&content_type=Article&match_order=1&q=条件变量&zd_token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJ6aGlkYV9zZXJ2ZXIiLCJleHAiOjE3NjAxNzE2MjAsInEiOiLmnaHku7blj5jph48iLCJ6aGlkYV9zb3VyY2UiOiJlbnRpdHkiLCJjb250ZW50X2lkIjoyNTcyMDA2MjQsImNvbnRlbnRfdHlwZSI6IkFydGljbGUiLCJtYXRjaF9vcmRlciI6MSwiemRfdG9rZW4iOm51bGx9.uUkOPn4MeTzNWWgI3nWLsHdwh7-3eXTXiyNgk-y3-4Y&zhida_source=entity)：`notEmpty_`和`notFull_`
- 实现背压机制，自动限流

**无界队列**

- **单向阻塞**：只有队列空时消费者会阻塞
- 只需一个条件变量：`notEmpty_`
- 没有自动限流机制

### 1.2.3 性能特性

**有界队列**

- 预分配内存可能提高缓存局部性
- 环形缓冲区实现可减少内存移动
- 添加元素时可能阻塞，总体吞吐量可能受限

**无界队列**

- 动态分配可能导致性能波动
- 添加元素几乎不会阻塞，高峰期吞吐量可能更高
- 可能导致系统内存压力增大

**examples/[http://test_blockingqueue_perf.cc](https://link.zhihu.com/?target=http%3A//test_blockingqueue_perf.cc)** 性能测试代码，当故意让 “生产者线程数 > 消费者线程数 ” 时，明显无界队列性能就更高。

**5个生产线程 6个消费线程**

|      | 无界队列 | 有界队列 |
| ---- | -------- | -------- |
| QPS  | 49.2万   | 41.3万   |

**5个生产线程 5个消费线程**

|      | 无界队列 | 有界队列 |
| ---- | -------- | -------- |
| QPS  | 44.4万   | 34万     |

**5个生产线程 3个消费线程**

|      | 无界队列 | 有界队列 |
| ---- | -------- | -------- |
| QPS  | 55.5万   | 23.2万   |

### 1.2.4 安全保障

**有界队列**

- 提供系统资源使用的上限保证
- 防止内存耗尽和系统失控
- 可能导致任务被阻塞或拒绝

**无界队列**

- 不提供资源使用上限保证
- 在极端情况下可能导致内存耗尽
- 几乎不会拒绝或丢弃任务

## 2 阻塞队列（无界/有界）实现

## 2.1 概述

阻塞队列是一个线程安全的数据结构，主要用于解决生产者-消费者问题。muduo提供了两种实现：

- 无界阻塞队列（BlockingQueue）
- 有界阻塞队列（BoundedBlockingQueue）

### 2.1.1 主要特点

1.线程安全

- 使用[互斥锁](https://zhida.zhihu.com/search?content_id=257200624&content_type=Article&match_order=1&q=互斥锁&zd_token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJ6aGlkYV9zZXJ2ZXIiLCJleHAiOjE3NjAxNzE2MjAsInEiOiLkupLmlqXplIEiLCJ6aGlkYV9zb3VyY2UiOiJlbnRpdHkiLCJjb250ZW50X2lkIjoyNTcyMDA2MjQsImNvbnRlbnRfdHlwZSI6IkFydGljbGUiLCJtYXRjaF9vcmRlciI6MSwiemRfdG9rZW4iOm51bGx9.TlB3KXt_dDj3u2GR6q6p7IAlvZMKC49mhWNFaySIH74&zhida_source=entity)保护共享数据
- 使用条件变量实现线程同步
- 支持多生产者多消费者模型

2.高性能

- 批量操作支持
- [移动语义](https://zhida.zhihu.com/search?content_id=257200624&content_type=Article&match_order=1&q=移动语义&zd_token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJ6aGlkYV9zZXJ2ZXIiLCJleHAiOjE3NjAxNzE2MjAsInEiOiLnp7vliqjor63kuYkiLCJ6aGlkYV9zb3VyY2UiOiJlbnRpdHkiLCJjb250ZW50X2lkIjoyNTcyMDA2MjQsImNvbnRlbnRfdHlwZSI6IkFydGljbGUiLCJtYXRjaF9vcmRlciI6MSwiemRfdG9rZW4iOm51bGx9.mAHxGo0x8OMgxHX5-RBjJx-tpgghyQvBpw8Fy1qwNtk&zhida_source=entity)优化
- 最小化锁竞争

## 2.2 详细设计

### 2.2.1 无界阻塞队列（BlockingQueue）

![img](https://pic4.zhimg.com/v2-745260bed0e7c90d40756b0925df9c5d_1440w.jpg)

### 核心成员

```text
template<typename T>
class BlockingQueue : noncopyable {
private:
    mutable MutexLock mutex_;    // 互斥锁
    Condition notEmpty_;         // 非空条件变量
    std::deque<T> queue_;       // 底层容器
};
```

### 关键方法实现

```text
// 放入元素（拷贝版本）
void put(const T& x) {
    MutexLockGuard lock(mutex_);
    queue_.push_back(x);
    notEmpty_.notify(); 
}

// 放入元素（移动版本）
void put(T&& x) {
    MutexLockGuard lock(mutex_);
    queue_.push_back(std::move(x));
    notEmpty_.notify();
}

// 取出元素
T take() {
    MutexLockGuard lock(mutex_);
    while (queue_.empty()) {
        notEmpty_.wait();
    }
    T front(std::move(queue_.front()));
    queue_.pop_front();
    return front;
}
```

### 2.2.1 有界阻塞队列（BoundedBlockingQueue）

![img](https://pic2.zhimg.com/v2-3851fc66dd055224595d2d7dae7e2591_1440w.jpg)

### 核心成员

```text
template<typename T>
class BoundedBlockingQueue : noncopyable {
private:
    mutable MutexLock mutex_;    // 互斥锁
    Condition notEmpty_;         // 非空条件变量
    Condition notFull_;          // 非满条件变量
    size_t capacity_;            // 队列容量
    std::vector<T> queue_;       // 存储元素的数组
    size_t head_;                // 队列头指针
    size_t tail_;                // 队列尾指针
    size_t size_;                // 当前队列元素个数
};
```

## 3.3 性能优化策略

1.批量操作

base/BlockingQueue.h 可以批量读取数据，减少生产/消费的锁竞争。

```text
// 批量取出所有元素
queue_type drain() {
 MutexLockGuard lock(mutex_);
 queue_type result;
 result.swap(queue_);
 return result;
}
```

2.减少锁竞争

```text
// 预分配内存减少扩容
queue_.reserve(initialCapacity);

// 使用双缓冲减少锁竞争时间
void putBatch(const std::vector<T>& batch) {
    MutexLockGuard lock(mutex_);
    for (const auto& item : batch) {
        queue_.push_back(item);
    }
    notEmpty_.notify_all();
}
```
