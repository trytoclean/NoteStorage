## 4 | 将清单转化为 12 个月节奏

1. **主题‑里程碑法**
   - 每季度聚焦 1 ‑ 2 个核心主题（如 Q1: 数据结构+算法 + Python 深度，Q2: 操作系统 + 网络）。
   - 每月安排一个可交付作品：小型开源库、博客系列、项目重构等。
2. **循环节奏**
   - **Week 1‑3**：学习+练习
   - **Week 4**：复盘+博客输出
   - **Week 5**：缓冲 / 知识迁移 / side project
3. **持续可见化**
   - 代码→GitHub学习笔记→博客阶段性成果→更新简历 / 领英
4. **评估仪表盘**
   - 每月用前述自评表快速打分一次 → 绘曲线看趋势。
   - 发现停滞就缩小目标或切换学习方式（做项目 > 只读书）。

## 5 | 常见疑问 & 建议

| 问题                       | 建议                                                         |
| -------------------------- | ------------------------------------------------------------ |
| 选 Go 还是 Python？        | 若你偏向后端/微服务、高并发，Go 更优；若脚本自动化、数据科学/AI，Python 更优。**先选一个专精**；半年后再考虑第二语言。 |
| CS 基础范围太大，怎么啃？  | 每门课只抓“骨架”概念 + 面试高频考点。利用 MIT OpenCourseWare / CSAPP·Lab / LeetCode 顶级题单来定位深坑。 |
| 一周完成会不会太赶？       | 这一周是**盘点**，不是补完。目标是识别盲区，而非立即填满。   |
| 如何保证 12 个月计划不崩？ | ① 保证 20 %‑30 % 缓冲时间；② 月度复盘时允许调整；③ 采用 **可交付物驱动** 而非纯时长驱动。 |
