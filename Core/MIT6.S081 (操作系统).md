# MIT6.S081 (操作系统)



### day one:

​	理解操作系统顶层设计：

目的：

- 实现抽象：让应用认为自己独占整个硬件资源。
- 访问控制：不同的用户应只能访问允许自己访问的资源。
- 发挥性能：与下层硬件交互，让硬件发挥性能。
- 运行安全：不同的进程可以同时运行，且互相不会影响彼此。

课程安排：最终实现一个XV6的操作系统。 每周都有实验。

用户态和内核态：用户态上运行诸如shell vim 等应用，内核态内运行如驱动等与硬件交互的程序。

所有进程只能在用户运行，一旦涉及到诸如IO等操作会通过系统调用，将当前状态保存（如寄存器，PC等），切换到内核态，每个应用进程有自己的文件描述符，通过这个文件描述符，与内核交互……



一些常见的系统调用；

​	open(fd,"loc",att)  打开文件

​	read，write，   读取，写入 文件

​	fork() 创建子进程，规定0为子进程，大于0为父进程

​	exec（） 执行命令    等等

一般小于0说明系统调用失败
